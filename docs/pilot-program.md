# Pilot Program

**Version:** Auroranexis v0.96  
**Duration:** 6 weeks  
**Capacity:** 3 pilot customers

---

## Program goals

1. Validate product-market fit with real agency workflows
2. Identify friction in onboarding, reliability, and UX
3. Prioritize post-pilot roadmap from customer feedback
4. Generate case study candidates

---

## Pilot structure

| Week | Focus |
|------|-------|
| 1 | Onboarding, workspace setup, first client imported |
| 2 | Reports + scheduling live |
| 3 | Risks, incidents, SLA |
| 4 | Automation + connectors |
| 5 | Compliance + API (if applicable) |
| 6 | Retrospective, feedback, go/no-go for production |

---

## What pilot customers receive

- Direct Slack/email support channel with product team
- Weekly 30-minute feedback call
- **50% discount** on first 6 months (see [pricing-beta.md](./pricing-beta.md))
- Feature prioritization input
- Early access to roadmap items
- Dedicated staging-to-production migration assistance

---

## What we ask from pilots

- Named product champion (owner or admin role)
- Weekly structured feedback (see [pilot-feedback.md](./pilot-feedback.md))
- Permission to use anonymized metrics for case studies
- Commitment to migrate at least 3 real clients into the platform
- Response within 48h on blocking issues

---

## Selection criteria

| Criterion | Target |
|-----------|--------|
| Agency size | 5–50 clients managed |
| Technical readiness | Owner comfortable with OAuth/Stripe |
| Use cases | Reports + client management minimum |
| Feedback culture | Willing to join weekly calls |

---

## Success metrics

| Metric | Target |
|--------|--------|
| Onboarding complete | < 3 days |
| Active weekly usage | 3+ sessions/week |
| Reports published | ≥ 1 per week by week 3 |
| NPS at week 6 | ≥ 40 |
| Critical bugs | 0 unresolved > 48h |

---

## Exit criteria

Pilot graduates to production when:

- [ ] 6-week program completed
- [ ] No P0/P1 bugs open
- [ ] Customer confirms production readiness
- [ ] Stripe live subscription active
- [ ] Production domain configured
