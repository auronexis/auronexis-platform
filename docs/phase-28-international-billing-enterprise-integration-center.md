# Phase 28 — International Billing & Enterprise Integration Center

**Date:** 2026-07-07  
**Status:** Complete  
**Branch:** `main`

## Executive Summary

Phase 28 prepares Auroranexis for international B2B customer acquisition by making billing UI fully localized (German/English), adding a permanent organization language setting, and converting the operational integrations experience into a production-grade Integration Center with real connection status — no placeholders or fake values.

Stripe-synchronized invoice PDFs remain generated by Stripe; this phase localizes all in-app invoice UI, formatting, and status labels. AI architecture, billing webhooks, and auth/RBAC were not changed.

## Architecture Decisions

1. **Centralized i18n module (`src/lib/i18n/`)** — All invoice strings live in `invoice.ts`. Locale resolution follows: organization language → customer language (future hook) → browser language → English fallback. Invoice components consume translations; no language-specific logic inside UI components.

2. **Organization language on `organizations.language`** — Persistent column (`de`|`en`, default `de`) rather than a separate preferences table. Reuses existing organization settings form and `updateOrganizationAction`.

3. **Integration Center as operational snapshot** — `getIntegrationCenterSnapshot()` aggregates real data from existing services (AI usage, connectors, Stripe env, API dashboard, webhook deliveries, report email deliveries). Missing data renders as **"No data available"**.

4. **OpenAI test connection** — Server action calls `createOpenAIProvider().health()` — same pattern as existing provider architecture. Visibility only; no AI generation or billing changes.

## Remaining Limitations

1. **Stripe invoice PDFs** — Generated by Stripe; language follows Stripe account/customer settings, not organization language.
2. **Slack connected channels** — Not tracked in database; displays "No data available".
3. **Resend domain verification** — No Resend domain API integration; verified status shows "No data available".
4. **Customer language** — Resolution hook exists; per-customer language not yet stored.
5. **Billing UI outside invoice panel** — Subscription/billing status labels in `status.ts` remain English (out of invoice scope).

## Recommendations for Phase 29

2. Add Resend domain verification status when API credentials allow.
